# Contributing to SharePlate

Thank you for considering contributing! This file explains how to set up the project locally and how to submit fixes or features.

1. Fork the repo and create a branch for your change:
```bash
git checkout -b feature/your-feature
```

2. Run the app and test your changes locally (see `README.md`).

3. Keep commits small and focused. Use clear commit messages.

4. Open a Pull Request describing the change, motivation, and any migration steps.

Coding style
- Use modern JavaScript (ES modules) and React hooks.
- Keep styles in Tailwind classes; avoid inline styles except for tiny exceptions.

Testing
- Manual testing steps are sufficient for UI changes in this project. Add unit tests where practical.
